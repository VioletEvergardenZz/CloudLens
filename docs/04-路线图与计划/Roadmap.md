# Roadmap

## 当前阶段

- 把云镜收敛成顺手的云服务器监控工具
- 稳定阿里云 ECS/RDS 与华为云 ECS/RDS 查询和监控视图
- 校准实例数据和监控指标的准确性
- 让文档、配置、部署说明一致

## 当前体检结论

- 测试门禁状态良好：后端单元测试、前端 lint 和前端生产构建均可通过。
- 云账号接入已经覆盖阿里云和华为云，但仍需要继续补真实账号样本，尤其是华为云 ECS/RDS 的 CES 指标命名和包年包月到期时间。
- 控制台已经能按 ECS/RDS、云平台、账号和地域组织资源，但刷新状态、空数据提示、局部失败提示仍应持续打磨。
- 阿里云和华为云 RDS 已接入列表、连接端点、空间用量和性能概览；下一步重点不是新增更多产品，而是把指标说明和异常提示做稳定。
- 告警、AI、知识库和文件入云保留为辅助能力，当前不宜单独扩张，应接到实例异常排查流程里。

## 当前阶段优先级

1. 稳定云账号和资源刷新链路，确保空账号、停用账号、局部权限不足和单指标失败都有可读反馈。
2. 补华为云真实账号回归样本，校准 ECS/RDS 字段、CES 指标单位和到期状态。
3. 强化监控视图的可解释性，明确每个指标的数据来源、单位和采样周期。
4. 补齐主线接口回归脚本，覆盖 `/api/health`、`/api/dashboard?mode=light`、云账号列表和云资源概览。
5. 文档跟随代码更新，尤其是本地开发端口、Docker 对外端口、云账号字段语义和权限清单。

## 已补强内容

- 新增 `scripts/ops/cloud-mainline-check.sh`，用于固定回归健康检查、降级仪表盘、云账号列表和已启用账号的阿里云/华为云 ECS/RDS 首个概览接口。
- 新增 `scripts/ops/huawei-real-account-check.sh`，用于在真实华为云账号下采集 ECS/RDS 字段、CES 指标单位和到期状态样本。
- 控制台资源同步失败已经按云平台、账号、资源类型和范围展示，便于区分“账号级失败”和“单资源监控失败”。
- 监控图表已经展示数据来源、原始指标和采样周期，减少指标理解偏差。
- 阿里云 ECS 指标查询已经统一采样周期兜底，并兼容云监控返回的 `timestamp`、`Timestamp`、`Date` 等采样时间字段，避免无效时间点影响监控新鲜度判断。
- 资源详情已经提供监控、告警、AI 摘要和知识库入口，作为实例异常排查联动的最小起点。
- Provider 抽象先沉淀到 `docs/02-架构设计/provider-抽象草案.md`，后续只从 ECS 只读共同字段开始落地。

## 下一阶段

目标是把 CloudLens 从“能看资源”推进到“能识别风险并辅助处理”，但仍然围绕轻量云服务器监控主线收口。

- 接入真实华为云账号后保存 `reports/huawei-real-account-check-*.json`，把样本结论回填到开发指南。
- 稳定阿里云 ECS/RDS 与华为云 ECS/RDS 指标口径，减少跨云单位、时间粒度和空数据误读。
- 建设资源风险视图，先覆盖健康分、即将到期、指标异常、监控缺失和 RDS 存储水位。
- 建设到期与成本风险提示，识别 7/15/30 天临期资源、长期低负载高规格实例、长期运行按量实例和疑似闲置实例。
- 强化告警事件闭环，支持分级、确认、处理中、已恢复、处理记录和恢复时间。
- 抽象统一 provider 接口时只从现有阿里云/华为云只读查询共同字段开始，不提前纳入写操作和复杂调度。
- 强化云资源与告警、知识库的联动时，优先做“从资源详情进入实例异常排查”的只读入口。
- 如果继续复用旧能力，就把文件入云、AI、知识库统一挂到实例异常排查链路，不恢复为独立产品主线。
- 后续再补巡检报告导出、通知集成和轻量安全基线检查，用于沉淀日报、周报、故障复盘和关键风险推送。
