# 开发指南

## 启动后端

```bash
cd cloudlens-backend
cp .env.example .env
go run ./cmd -config config.yaml
```

如果你在仓库根目录 `CloudLens/` 下：

```bash
go run ./cloudlens-backend/cmd -config ./cloudlens-backend/config.yaml
```

默认行为：

- 没有 OSS 也能启动
- 没有 AI 也能启动
- 没有 `watch_dir` 也能启动
- 默认使用 SQLite
- 系统资源监控默认关闭，当前主线不依赖它

## 启动前端

```bash
cd cloudlens-frontend
npm install
npm run dev
```

## Docker Compose

```bash
cp .env.example .env
docker compose up --build -d
```

当前建议：

- 继续用 SQLite
- 只把 `config/`、`logs/`、`data/` 挂出来
- 现在不用单独起数据库容器

## 如果你要走云监控主线

至少准备：

```bash
ALIYUN_ACCESS_KEY_ID=your-ak
ALIYUN_ACCESS_KEY_SECRET=your-secret
ALIYUN_REGION=cn-hangzhou
```

当前真实落地以阿里云单账号 ECS 为主。

## 如果你还要用文件入云

再补：

```bash
OSS_BUCKET=your-bucket
OSS_ENDPOINT=https://oss-cn-hangzhou.aliyuncs.com
OSS_REGION=cn-hangzhou
OSS_AK=your-ak
OSS_SK=your-sk
```

建议不要再把这条能力当作产品主入口，后续更适合挂到实例日志采样或异常证据归档场景。

## 测试命令

```bash
cd cloudlens-backend
go test ./... -count=1
```

```bash
cd cloudlens-frontend
npm run lint
npm run build
```
