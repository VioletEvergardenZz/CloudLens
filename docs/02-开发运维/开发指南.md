# 开发指南

## 启动后端

```bash
cd cloudlens-backend
cp .env.example .env
go run ./cmd
```

如果你在仓库根目录 `CloudLens/` 下：

```bash
go run ./cloudlens-backend/cmd
```

默认行为：

- 不指定配置文件也能启动
- 默认读取内置配置、环境变量和当前工作目录下的 `.env`
- 没有 OSS 也能启动
- 没有 AI 也能启动
- 没有 `watch_dir` 也能启动
- 默认使用 SQLite
- 系统资源监控默认关闭，当前主线不依赖它
- 如需加载旧文件入云静态配置，可显式传入 `-config config.yaml`

## 启动前端

```bash
cd cloudlens-frontend
npm install
npm run dev
```

本地开发时后端默认监听 `:8080`，Vite 会把 `/api` 代理到 `http://localhost:8080`。Docker Compose 场景下后端容器内仍监听 `8080`，但默认对宿主机暴露为 `http://localhost:8082`。

## Docker Compose

```bash
cp .env.example .env
docker compose up --build -d
```

当前建议：

- 继续用 SQLite
- 只把 `config/`、`logs/`、`data/` 挂出来
- 现在不用单独起数据库容器

## 如果你要走云监控主线

至少准备：

```bash
ALIYUN_ACCESS_KEY_ID=your-ak
ALIYUN_ACCESS_KEY_SECRET=your-secret

HUAWEI_ACCESS_KEY_ID=your-ak
HUAWEI_ACCESS_KEY_SECRET=your-secret
```

当前真实落地以阿里云 ECS/RDS 和华为云 ECS/RDS 为主。也可以直接在控制台“云账号”页面新增多个阿里云或华为云账号；新增账号不需要选择地域，也不需要填写华为云 Project ID，后端会按账号自动发现并采集全部可见地域，资源列表仍按云厂商返回的地域分组展示。

编辑云账号时，AccessKey ID 和 Secret 留空表示沿用原凭据。

如果要查看多个云账号下的 RDS，先在“云账号”页面确认这些阿里云或华为云账号已启用；然后进入“资源总览”，展开“RDS -> 云平台 -> 账号 -> 地域”，按资产类型、云平台、账号和地域逐层查看。表格顶部也可以选择“只看 RDS”，用于跨账号过滤，避免 ECS 和 RDS 混在一起。

`GET /api/cloud/aliyun/instances` 会返回 ECS 到期信息：`expiredAt` 是云厂商原始到期时间，`expiresInDays` 是不高估的完整剩余天数，`expirationStatus` 用于区分正常、30 天内到期、已到期、无固定到期日和未知状态。按量付费与抢占式实例会通过 `chargeType`、`isSpot`、`spotStrategy` 标注，并视为无固定到期日；云厂商返回的远期占位时间也会按无固定到期日处理。

`GET /api/cloud/aliyun/rds/instances` 会返回 RDS 实例基础信息、规格、存储、连接端点、到期状态和 `DescribeResourceUsage` 官方空间用量。`GET /api/cloud/aliyun/rds/overview` 会按 RDS 引擎分批查询性能参数；单个性能 Key 不支持时只进入 `errors`，不影响其它指标展示。RAM 账号需要具备 `AliyunRDSReadOnlyAccess`，否则前端会保留 ECS 数据并提示 RDS 局部同步失败。

`GET /api/cloud/huawei/rds/instances` 会返回华为云 RDS 实例基础信息、节点 ID、规格、存储、连接端点、到期状态和 `ShowStorageUsedSpace` 官方空间用量。`GET /api/cloud/huawei/rds/overview` 会从 CES `SYS.RDS` 查询 CPU、内存、QPS、连接数和 IOPS；单个指标无数据或失败只进入 `errors`，不影响其它指标展示。IAM 账号需要具备 RDS 只读和 CES 云监控只读权限，否则前端会保留 ECS 数据并提示 RDS 局部同步失败。

阿里云 ECS 监控概览优先展示云厂商真实采样：CPU、内存、磁盘使用率按 `%` 展示；公网/内网流量统一按 `bit/s` 展示；磁盘读写吞吐按 `Byte/s` 展示。`DescribeInstanceMonitorData` 返回的 `InternetRX/InternetTX/IntranetRX/IntranetTX` 是采样周期内的 Kbit 流量累计值，后端会按周期换算为 `bit/s`，避免前端把累计量误当瞬时速率。

阿里云 ECS 监控采样周期会统一兜底为有效秒数，未填写或填写非法值时使用 `60` 秒。云监控返回的采样时间兼容 `timestamp`、`Timestamp`、`Date` 等字段；没有有效时间或数值的采样点会被忽略，避免页面把异常数据误判为 1970 年旧采样。

RDS 列表中的磁盘百分比优先使用云厂商官方空间用量接口返回的数据，并按“官方已用空间 / 实例存储容量”换算；阿里云使用 `DescribeResourceUsage`，华为云使用 `ShowStorageUsedSpace`。如果官方空间接口不可用，再回退到性能参数中的百分比或 Byte/KB/MB/GB 空间值，避免把 20GB、102400MB 这类容量值误当成 100%。

监控概览使用“左侧资源选择器 + 右侧图表”的工作区布局；探针管理使用“全局覆盖概览 + 状态泳道 + 账号/地域/来源治理”的工作台布局。探针管理的可用采样覆盖率只统计最新可用且有真实采样的资源；采样点、最近采样和采样来源统计只基于真实监控采样点，没有真实采样时显示“暂无真实采样”。

`GET /api/cloud/huawei/instances` 返回的 ECS 字段与阿里云 ECS 对齐，包括实例 ID、名称、地域、可用区、规格、CPU、内存、公网/内网 IP、VPC、子网、安全组、创建时间、计费类型和到期状态。华为云 ECS 详情接口未返回包年包月到期时间时，`expirationStatus` 会返回 `unknown`，页面会展示“待云厂商返回”。

`GET /api/cloud/huawei/overview` 会从 CES 查询 CPU、内存、磁盘、负载、网络和磁盘吞吐指标。CES datapoint 返回单位时以后端返回的单位为准；公网/内网字节速率在概览接口中换算为 `bit/s` 并同步标注单位，磁盘吞吐保持 `Byte/s`，CPU、内存和磁盘使用率保持 `%`。

前端“监控概览”和“探针管理”每 30 秒自动刷新一次云账号、资源和监控采样。监控概览右侧会展示资源概况、指标说明、采样来源和图表；探针管理顶部展示采样覆盖率、治理待处理、真实采样点和最近采样时间，中部按正常、数据异常、未同步、离线组织状态泳道，下方按治理提醒、账号覆盖、地域缺口和采样来源质量聚合展示。

## 主线回归脚本

轻量回归：

```bash
cd cloudlens-backend
scripts/ops/cloud-mainline-check.sh --base-url http://localhost:8082 --output-file ../reports/cloud-mainline-check-$(date +%F).json
```

这个脚本会固定检查 `/api/health`、`/api/dashboard?mode=light`、`/api/cloud/accounts`。如果控制台已经保存并启用了云账号，还会抽样检查阿里云 ECS/RDS、华为云 ECS/RDS 的列表和首个监控概览接口。没有云账号时云资源抽样会标记为 skipped，但健康检查和降级仪表盘仍会参与通过判定。

华为云真实账号回归：

```bash
cd cloudlens-backend
scripts/ops/huawei-real-account-check.sh --base-url http://localhost:8082 --output-file ../reports/huawei-real-account-check-$(date +%F).json
```

也可以用 `--account-id` 指定某个华为云账号。脚本会输出 ECS/RDS 样本字段、CES 指标单位、采样周期、指标错误和到期状态。没有真实华为云账号时脚本会明确失败并提示先新增或启用账号，不会伪造样本。

## 如果你还要用文件入云

再补：

```bash
OSS_BUCKET=your-bucket
OSS_ENDPOINT=https://oss-cn-hangzhou.aliyuncs.com
OSS_REGION=cn-hangzhou
OSS_AK=your-ak
OSS_SK=your-sk
```

建议不要再把这条能力当作产品主入口，后续更适合挂到实例日志采样或异常证据归档场景。
