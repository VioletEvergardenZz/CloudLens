# app 目录地图

`app/` 是应用服务层，用来承接跨 handler、跨存储、跨云厂商的业务编排。这里的代码应该表达“产品要做什么”，不直接暴露 HTTP 细节，也不直接散落云厂商 SDK 细节。

## 当前模块

| 目录 | 作用 |
| --- | --- |
| `cloud/` | 统一资源索引、巡检风险和 K8s Node 关联 |

## 维护规则

- handler 变厚时，优先把可复用业务编排下沉到 `app/`。
- `app/` 可以调用 `store/` 和云资源领域类型，但不要依赖 HTTP 请求对象。
- 云厂商 SDK 调用仍放在 `internal/cloud/{provider}`，不要直接写进 `app/`。
