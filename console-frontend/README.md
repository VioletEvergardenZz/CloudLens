# 统一运维控制台（console-frontend）

> 文档状态：组件文档（前端）  
> 校准日期：2026-02-24
> 口径说明：前端行为说明需与后端接口和 docs/ 主线文档一致。

`console-frontend` 是 GWF 平台的统一操作入口，负责把“文件入云 + 告警决策 + AI 分析 + 知识库复用”组织成可执行闭环，而不是只做文件上传看板。

## 在平台中的定位

- 承载多视图控制台：
  - 平台总览
  - 主控制台（文件与运行态总览）
  - 告警控制台
  - 事件工作台（原型）
  - 系统资源控制台
  - 域名接入控制台
  - 系统台账（原型）
  - 运维知识库控制台
  - 控制面控制台（Agent/Task/Audit）
- 统一对接 `go-watch-file` 提供的 API。
- 提供值班可用的实时刷新、筛选和处置入口。

## 一级导航与工作域

当前控制台不再按“技术模块”组织一级导航，而按值班实际工作流组织为 6 个工作域：

1. 值班：平台总览与当班入口
2. 事件：告警中心与事件工作台
3. 接入：文件接入与域名接入
4. 系统：系统台账与系统详情
5. 知识：知识检索、沉淀、审核与复用
6. 执行：控制面、任务追踪与审计

这样做的目标是把“发现 -> 分诊 -> 处置 -> 复盘 -> 沉淀”收拢成一条主线，而不是让用户在多个孤立控制台之间来回跳转。

## 主要能力（与当前代码一致）

- 仪表盘：目录树、上传记录、队列趋势、文件检索、运行时配置。
- 总览：全局状态、待办事项、最近变化、主线运行健康。
- 告警：告警概览、决策列表、规则与配置管理、知识推荐联动。
- 事件工作台：聚合单事件时间线、分析区、知识与控制面动作入口。
- 系统资源：CPU/内存/磁盘/进程视图与进程终止操作。
- 域名接入：域名入口探测、候选健康接口识别、待确认事项生成、结果导出。
- 系统台账：系统 / 环境 / 服务 / 路由 / 健康规则原型视图。
- 知识库：条目管理、检索、问答、审核与回滚流程。
- 控制面：Agent 状态、任务队列、任务事件、审计日志、取消/重试操作。

## 快速开始

```bash
cd console-frontend
npm install
npm run dev
```

默认通过 Vite 将 `/api` 代理到 `http://localhost:8080`。  
若后端地址不同，可设置 `VITE_API_BASE`。

## 接口访问

- 当前版本不需要在前端配置 API Token。
- 控制台请求默认直连后端接口；如需跨域限制，请在后端配置 `API_CORS_ORIGINS`。

## 目录结构

- `src/App.tsx`：应用入口与视图路由。
- `src/OriginalConsole.tsx`：主控制台编排。
- `src/OverviewConsole.tsx`：平台总览页。
- `src/AlertConsole.tsx`：告警控制台。
- `src/EventWorkbench.tsx`：事件工作台原型。
- `src/SystemConsole.tsx`：系统资源控制台。
- `src/RegistryConsole.tsx`：域名接入控制台。
- `src/RegistryCatalogConsole.tsx`：系统台账原型。
- `src/KnowledgeConsole.tsx`：知识库控制台。
- `src/ControlConsole.tsx`：控制面控制台。
- `src/console/dashboardApi.ts`：前后端 API 封装。
- `src/types.ts`：统一类型定义。
